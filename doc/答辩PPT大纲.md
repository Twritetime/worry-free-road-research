### 第3页：选题背景
- **考研热潮持续升温**：2024年考研报名人数达438万，备考信息获取需求旺盛
- **现有平台痛点**：
  - 信息分散，缺乏一站式服务平台
  - 资料质量参差不齐，难以精准匹配
  - 缺乏个性化学习规划与智能答疑
- **AI技术赋能教育**：大模型技术为个性化教育服务提供新可能

### 第4页：选题意义
- **理论意义**：探索AI大模型在教育服务领域的应用模式，为智能教育平台设计提供参考
- **实际意义**：
  - 为考研学生提供"资讯+资料+社区+AI助手"一站式服务
  - 降低信息获取成本，提高备考效率
  - 通过AI智能推荐实现个性化学习路径规划
  - 构建活跃的考研交流社区，形成学习互助生态

### 第6页：系统需求分析 - 功能模块

  - 资料商城：分类浏览、搜索筛选、购物车、在线支付
  - AI智能助手：智能问答、资料推荐、学习计划生成

  - 备考指南：经验分享、科目复习建议
- **后台管理端**：
  - 数据统计仪表盘、用户管理、内容管理、订单管理、反馈管理

### 第8页：系统总体设计 - 架构设计
- **分层架构**：
  - 表现层：Vue3前端 + Element Plus UI
  - 控制层：RESTful API控制器
  - 业务层：Service接口与实现
  - 数据层：MyBatis Plus Mapper + MySQL
- **部署架构**：
  - 前端：Vite构建，Nginx部署
  - 后端：Spring Boot内嵌Tomcat
  - 数据库：MySQL 8.0
  - 第三方：DeepSeek API、支付宝沙箱
- **图片预留位**：系统架构图 / 部署拓扑图

### 第9页：数据库设计 - ER图与核心表
- **核心实体关系**：
  - 用户（sys_user）→ 订单（yl_order）→ 订单项（yl_order_item）
  - 用户 → 购物车（yl_cart_item）→ 资料（yl_material）
  - 用户 → 帖子（yl_post）→ 评论（yl_comment）
  - 用户 → 收藏（favorite）、反馈（feedback）
  - 用户 → AI对话记录（chat_message）
- **关键设计**：
  - 逻辑删除（deleted字段）
  - 统一基础字段（create_time, update_time）
  - 角色权限控制（role字段：STUDENT/OPERATOR/ADMIN）
- **图片预留位**：数据库ER图 / 核心表关系图

### 第10页：核心模块实现 - AI智能助手
- **模块概述**：基于DeepSeek大模型的考研智能问答系统
- **核心实现**：
  - 系统Prompt工程：限定AI回答范围，禁止编造信息
  - 对话上下文管理：维护session会话历史
  - 资料关联推荐：AI回答后自动匹配相关学习资料
- **代码亮点**：
  - 动态温度系数调节（temperature=0.7）
  - 流式响应处理（可选）
  - 用户行为记录用于后续推荐优化
- **图片预留位**：AI对话界面截图 / Prompt设计代码片段截图

---

### 第11页：核心模块实现 - 智能推荐与支付
- **AI智能推荐**：
  - 基于用户行为数据（浏览、收藏、购买）构建用户画像
  - 调用DeepSeek API分析用户偏好，生成个性化推荐理由
  - 多场景推荐：热门推荐、猜你喜欢、阶段化推荐
- **支付宝支付流程**：
  - 订单创建 → 调用支付宝沙箱 → 用户扫码支付 → 异步通知回调 → 订单状态更新
  - 支持购物车批量结算与立即购买两种模式
- **图片预留位**：推荐算法流程图 / 支付时序图

---

### 第12页：核心模块实现 - 权限与安全
- **JWT认证机制**：
  - 登录成功后签发Token（有效期72小时）
  - 请求拦截器验证Token有效性
  - 自定义注解@RequireLogin、@RequireRoles控制访问权限
- **安全防护措施**：
  - 密码BCrypt加密存储
  - 内容安全过滤（防XSS攻击）
  - 接口限流保护（RateLimit）
  - 跨域配置（CORS）
- **角色权限矩阵**：
  - STUDENT：前台功能
  - OPERATOR：内容管理+订单管理
  - ADMIN：全部权限
- **图片预留位**：认证流程图 / 权限控制代码截图

### 第14页：创新点与总结
- **创新点**：
  1. **AI深度集成**：基于DeepSeek大模型实现智能问答、个性化推荐、学习计划生成三大AI功能
  2. **双端权限体系**：前台用户与后台管理员独立认证，支持STUDENT/OPERATOR/ADMIN三级角色
  3. **智能资料推荐**：结合用户行为数据与AI分析，实现"千人千面"的个性化资料推荐
  4. **一站式服务平台**：整合资讯、商城、社区、AI助手于一体，覆盖考研备考全场景
- **总结**：平台有效解决了考研学生信息分散、资料难选、缺乏规划的痛点，AI技术的应用显著提升了用户体验
- **展望**：未来可引入RAG技术增强AI回答准确性，接入更多大模型实现多模态交互
